 Data Exploration with Microsoft SQL Server
This project showcases a structured approach to exploratory data analysis (EDA) using Microsoft SQL Server, combining strong querying fundamentals with analytical thinking rooted in real-world financial and business contexts.

🔍 What This Project Covers
The exploration focuses on uncovering patterns, anomalies, and insights hidden within relational datasets using pure SQL. Techniques applied include:

Data profiling — assessing row counts, null distributions, and column cardinality
Aggregations & groupings — summarizing trends across dimensions such as time, category, and region
Filtering & segmentation — isolating meaningful subsets for focused analysis
Joins & subqueries — combining tables to build richer, multi-dimensional views
Window functions — ranking, running totals, and period-over-period comparisons
Data quality checks — identifying duplicates, outliers, and type inconsistencies

🛠️ Tools & Environment

Microsoft SQL Server (T-SQL)
SQL Server Management Studio (SSMS)
Queries designed for readability, reusability, and performance

💡 Purpose & Goals
The goal of this project is to demonstrate how structured querying can replace or complement traditional spreadsheet analysis — making it easier to work with larger datasets, automate recurring checks, and build a foundation for dashboards and reporting tools like Power BI.
This project is part of a broader journey transitioning domain expertise in financial services into a full data analytics skill set — bridging the gap between business intuition and technical execution.
